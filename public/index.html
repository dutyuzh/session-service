🔐 Session Service API Tester

Local demo with Redis • Interactive testing interface for on-prem session management

⚠️ Requires local Redis server
📚 View Architecture Documentation

Step 1: Create Session

POST /sessions

Create a new anonymous session and receive a session cookie.

Step 2: Fetch Session

GET /sessions/me

Retrieve your current session data using the session cookie.

Step 3: Update Activity

POST /sessions/me/activity

Ping activity to refresh the sliding 30-minute expiration window.

Step 4: Update Attributes

PATCH /sessions/me

Attach custom metadata like campaignId, referralSource, etc.

Step 5: Device Identity

GET /sessions/me

Check your device ID (did cookie). Device persists 90 days across browser sessions.

Device ID: Loading...

Step 6: Attach Identity

POST /sessions/me/attach-identity

Transition from anonymous to authenticated. Links userId to session and device.

Step 7: Restore Session

GET /sessions/me

Test persistent login: Delete session cookie and fetch session. If device has userId, new session auto-restores!

⚠️ Prerequisites: You must first complete Step 6 (Login / Attach Identity) to give your device a userId. Anonymous sessions cannot be restored.
Status: ...

Step 8: List Devices

GET /users/me/devices

View all devices linked to your user. Test multi-device: Login from different browsers!

Step 9: Merge Sessions

POST /sessions/merge

Merge two sessions into one. Attributes merged, target wins on collisions. Lineage preserved!

Step 10: Invalidate Session

POST /sessions/me/invalidate

Explicitly logout and destroy session. Cookie cleared. Security: Step 6 now rotates sid on login (fixation mitigation)!

Step 11: Cross-Service Continuity

GET /internal/sessions/:sid

Requirement: "Sessions must be retrievable and usable across multiple services."
Tests internal API for microservices (email, billing, analytics) to access session data with shared secret authentication.

GET /health
